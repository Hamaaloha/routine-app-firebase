# 研究ルーティン PWA｜アプリ詳細

## 概要

日課・週次棚卸・研究計画をいつでも確認・記録できるWebアプリ（PWA）。
スマホ・PCのブラウザで動作し、オフライン対応・ホーム画面へのインストールが可能。

---

## ファイル構成

```
routine-app/
├── index.html      メインアプリ（HTML/CSS/JS をすべて内包）
├── manifest.json   PWAインストール設定
├── sw.js           サービスワーカー（オフラインキャッシュ）
└── README.md       このファイル
```

---

## 機能一覧

### ☀️ 今日タブ
- 今日の日付・曜日・平日/休日の自動判定
- **日課チェックリスト**（タップで完了/未完了を切り替え）
  - 新聞読み・論文執筆・論文読み
  - 平日と休日で内容を自動切り替え
- **進捗バー**（完了数 / 全習慣数 をパーセントで表示）
- 今週の優先研究（棚卸で保存した順位を反映）
- 次のマイルストーン表示
- チェック状態は LocalStorage に日付ごとに保存。翌日は自動リセット

### 📋 棚卸タブ
- 5ステップの週次棚卸フォーム
  - STEP 1: 先週の数値振り返り（執筆時間・論文本数・新聞日数）
  - STEP 2: 各研究の進捗テーブル（先週やったこと・今週のタスク）
  - STEP 3: 今週の優先順位（1〜4位）
  - STEP 4: 今週読む論文リスト（追加ボタンで行を増やせる）
  - STEP 5: 曜日別の執筆割り当て
- **保存ボタン**で LocalStorage に保存
  - `review_YYYY-M-D` キーで日付ごとに記録
  - `review_latest` キーで最新を保持（今日タブの優先順位に反映）

### 📊 計画タブ
- 研究ステータスカード（urgent/通常/完了で色分け）
- マイルストーン一覧（次の締切をハイライト）
- 大きな目標 → 週次ゴール の構造図

### 📖 ルールタブ
- 平日・休日のルーティン表
- 論文執筆の曜日割り当て
- 論文読みのルール
- 新聞読みの目的
- フェーズ変化への対応方針（就職後の調整）

## データの保存先

すべてのデータは**ブラウザの LocalStorage** に保存される。
サーバーへの送信は一切なし。

| キー | 内容 |
|------|------|
| `habits_YYYY-M-D` | その日の日課チェック状態 |
| `review_YYYY-M-D` | その日の棚卸フォームの入力内容 |
| `review_latest` | 最新の棚卸データ（今日タブの優先順位に使用） |

> ブラウザのデータを削除すると入力内容も消える。
> 重要な棚卸内容は weekly_research_plan.md にも転記しておくことを推奨。

## 使い方

### PCで開く
1. `index.html` をダブルクリック（Chromeまたは Edge で開く）
2. アドレスバー右端のインストールアイコン → 「インストール」でデスクトップアプリとして使える

### スマホでホーム画面に追加（PWAとして使う）
PWAのフルインストール（ホーム画面追加・オフライン対応）には **HTTPS でのホスティングが必要**。

#### 方法1：GitHub Pages（無料・推奨）
1. GitHubアカウントを作成（または既存アカウントを使用）
2. 新しいリポジトリを作成（例：`routine-app`）
3. `routine-app/` フォルダ内のファイルをすべてアップロード
4. Settings → Pages → Source を `main` ブランチに設定
5. `https://[ユーザー名].github.io/routine-app/` にアクセス
6. iOS: 共有ボタン → 「ホーム画面に追加」
7. Android (Chrome): メニュー → 「ホーム画面に追加」

#### 方法2：Netlify（無料・最も簡単）
1. [netlify.com](https://netlify.com) にアクセス
2. `routine-app/` フォルダをブラウザにドラッグ&ドロップ
3. 自動的にURLが発行される（例：`https://xxxxxx.netlify.app`）
4. そのURLからホーム画面に追加

#### ホスティングなしで使う場合
- PCのブラウザで `index.html` を直接開けば全機能が動作する
- スマホでは OneDrive モバイルアプリから `index.html` を開くことができる
  （ホーム画面追加はできないが、閲覧・入力・保存は可能）

## 研究データの更新方法

研究ステータス・マイルストーンなどのデータは `index.html` 内の JavaScript に直接記述されている。

### 研究ステータスを変更する場合
`index.html` を開き、以下の `RESEARCH` 配列を編集する（約60行目付近）：

```javascript
const RESEARCH = [
  { name:'野球（JSSC）', status:'本文完成・引用肉付け中', deadline:'2026/4/30', urgent:true },
  { name:'脳震盪',       status:'Results確定・Discussion強化中', deadline:'2026/5月中旬' },
  // ...
];
```

- `urgent: true` → 赤枠カード
- `done: true`   → グレーアウトカード
- `urgent/done なし` → 緑枠カード（通常）

### マイルストーンを変更する場合
`MILESTONES` 配列を編集する：

```javascript
const MILESTONES = [
  { date:'4/30', label:'野球（JSSC）投稿', next:true },  // next:true → 赤でハイライト
  { date:'3/20', label:'Mac3D完了', done:true },          // done:true → 完了扱い
  // ...
];
```
